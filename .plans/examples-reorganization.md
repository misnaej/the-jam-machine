# Plan: Examples Reorganization

**Date:** 2026-03-24
**Status:** Planned

---

## Goal

Create a well-organized `examples/` directory that demonstrates the two core use cases of The Jam Machine:

1. **Encoding/Decoding** — Take a real MIDI file (The Strokes - Reptilia), encode it to text tokens, then decode it back to MIDI
2. **Generation** — Generate new music from scratch using the GPT-2 model (existing script)

## Steps

### Step 1: Create `examples/encode_decode.py`

A self-contained script demonstrating the full roundtrip:

```python
"""Example: Encode a MIDI file to text tokens and decode back to MIDI.

Demonstrates the encoding/decoding pipeline using The Strokes - Reptilia
as a reference MIDI file. This is the same pipeline used to prepare
training data for the GPT-2 model.

Usage:
    pipenv run python examples/encode_decode.py
"""
```

The script should:
1. Load `midi/the_strokes-reptilia.mid`
2. Encode to text tokens using `MIDIEncoder` (with familized=True)
3. Print a preview of the encoded text (first 200 chars + stats)
4. Save encoded text to `output/examples/encode_decode/`
5. Decode the text back to MIDI using `TextDecoder`
6. Save the decoded MIDI to the same output directory
7. Generate and save a piano roll visualization
8. Log a summary comparing original vs decoded

`main()` accepts an optional `output_dir` parameter (defaults to `output/examples/encode_decode/`) so tests can redirect to `tmp_path`.

## Design Decisions

### Output directory

Both example scripts write to `output/examples/` (gitignored). This directory is:
- Created automatically by the scripts
- Documented in `examples/README.md` so users know where to find results
- Added to `.gitignore`

The generation example currently writes to `midi/generated/` — update it to use `output/examples/generation/` for consistency. The encode/decode example writes to `output/examples/encode_decode/`.

### Testing strategy

Tests import and call `main()` like today, but example scripts should accept an optional `output_dir` parameter so tests can pass `tmp_path` instead of writing to `output/examples/`. This keeps tests hermetic while letting users run the scripts directly and find output in a predictable location.

### Why not move MIDI files into `examples/data/`?

The `midi/` directory is already established and referenced by tests. Moving it would break the test fixtures and add unnecessary churn. The example script can reference `midi/` by relative path.

### Why keep `__init__.py` in examples?

Only needed if `test_examples.py` imports from `examples`. If we switch the test to use `subprocess.run` or `runpy.run_module`, we can delete it. Decide during implementation.
